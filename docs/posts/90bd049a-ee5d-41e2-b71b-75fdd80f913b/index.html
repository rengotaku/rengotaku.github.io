railsプロジェクトの高速化

April 23, 2019

設定ファイル#

version: '2'
services:
  rails:
    build: front.door.ac/
    command: >
      /bin/sh -c
      "rm -f /var/tmp/pids/server.pid
      && bundle exec rails s"      
    env_file: .env
    volumes:
      - .:/var
      - bundle:/usr/local/bundle:cached
      # exclude volumes
      - /var/vendor
      - /var/tmp
      - /var/log
      - /var/.git
    ports:
      - 3000:3000
    stdin_open: true
    tty: true

マウントが確認できない場合は、docker-compose down を実行して、コンテナを削除する。

設定で聞けつること#

ローカルで $ tail -f log/development.log できなくなる。 $ docker-compose rails tail -f log/development.log のようにdocker内にアクセスする必要がある。
